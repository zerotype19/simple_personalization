# Optiview — 5-minute demo script

Repeatable walkthrough for **prospects and pilot stakeholders** on the hosted demo. Read this aloud while driving the screen.

| Resource | URL |
|----------|-----|
| **Live demo** | [https://demo.optiview.ai](https://demo.optiview.ai) |
| Marketing | [https://optiview.ai](https://optiview.ai) |
| Privacy (buyer-safe) | [https://optiview.ai/privacy](https://optiview.ai/privacy) |
| Install (webmaster) | [CUSTOMER_INSTALL.md](./CUSTOMER_INSTALL.md) · [WEBMASTER_INSTALL_ONE_PAGER.md](./WEBMASTER_INSTALL_ONE_PAGER.md) |
| Security / privacy FAQ | [SECURITY_PRIVACY_FAQ.md](./SECURITY_PRIVACY_FAQ.md) |
| Pilot packaging | [PILOT_ONBOARDING.md](./PILOT_ONBOARDING.md) · [PILOT_SUCCESS_SCORECARD.md](./PILOT_SUCCESS_SCORECARD.md) |

**Story spine (say this once, then prove it on screen):**

> Interest deepens.  
> Reassurance may be needed.  
> Escalation must be earned.  
> Optiview explains the judgment **before** identity exists.

**Format:** **SAY** = exact or near-exact narration · **DO** = on-screen action · **POINT** = where to look

**Before the call (2 min):** Normal browser window (not Incognito) on `https://demo.optiview.ai`. Optional: click **New session · re-roll A/B** on the demo strip for a fresh journey.

---

## 1. Opening framing — 30 seconds

**SAY:**

> “Optiview is an **anonymous experience decision runtime**. It does **not** identify the visitor. It watches **commercial behavior** on your site and recommends **what experience should happen next** — which surface to show, when to wait, and when to hold back.  
> The difference from most personalization: Optiview also knows **when not to interrupt**.”

**DO:** Land on the demo home. Glance at the **Demo path progress** strip (under the nav) without dwelling on it yet.

**Do not say:** “AI reads the page,” “we personalize everything,” “we know who they are,” “we auto-popup,” “ChatGPT for your site.”

---

## 2. Show the page — 30 seconds

**SAY:**

> “This is **not** a full retailer site — it’s a **controlled journey** to show the runtime’s judgment. The strip labeled **Demo path progress** is the **scripted path** we’re walking. The **inspector** is Optiview’s **live runtime read** on this anonymous session.”

**POINT:** **Demo path progress** → Exploring → Evaluating → Comparing → Test-drive step. Helper copy: *“This shows where you are in the scripted demo path. The inspector shows Optiview’s runtime judgment.”*

**Do not say:** “The strip and inspector always match.” They are **two ladders**: path progress ≠ permission to escalate.

---

## 3. Run the journey — 2 minutes

Run the **core path** in order. After each step, glance at the on-page **live read** panel if visible, then move on — save the deep dive for the inspector in section 4.

### Step A — Compare vehicles (~30 sec)

**DO:** Go to **Compare**. Select **two** models in the shortlist.

| | |
|--|--|
| **What the visitor did** | Side-by-side comparison — narrowing options. |
| **What Optiview inferred** | Comparison behavior; movement from browsing into deliberate evaluation. |
| **Why it matters** | Comparison is commercial signal, not a page-view counter. The runtime favors **comparison support** over a hard escalation. |

**SAY (short):** “They’re narrowing — Optiview stays patient. Comparison support beats a premature dealer push.”

---

### Step B — Review financing options (~30 sec)

**DO:** Open **Finance** (e.g. **See payments for picks**). Adjust a payment slider or monthly inputs.

| | |
|--|--|
| **What the visitor did** | Validating payment and monthly cost while options are still open. |
| **What Optiview inferred** | Growing financing interest **with** ongoing comparison / validation. |
| **Why it matters** | **High interest does not automatically mean interrupt harder.** Reassurance can be the right next experience. |

**SAY (short):** “Interest is deepening, but they may still be validating financing. Optiview can favor **payment reassurance** before a stronger ask.”

**POINT (if shown):** On-page restraint note — growing interest, but the runtime held back a stronger dealer escalation.

---

### Step C — Book a test drive (~30 sec)

**DO:** Go to **Test drive**. Click through to the scheduling step (do **not** submit yet unless you want to save time).

| | |
|--|--|
| **What the visitor did** | Moved toward human contact — a high-intent structural signal. |
| **What Optiview inferred** | Strengthening contact intent alongside prior comparison and financing context. |
| **Why it matters** | Test-drive guidance can appear when readiness supports it — still balanced against open financing questions. |

**SAY (short):** “This is a strong move toward in-person next step — but the runtime still weighs **readiness**, not just eagerness.”

---

### Step D — Submit test-drive request (~30 sec)

**DO:** Submit the form (notes optional). Dismiss any alert. Let the **inspector** open in **Buyer view** if it auto-opens.

| | |
|--|--|
| **What the visitor did** | Submitted a scheduling-intent form. |
| **What Optiview inferred** | Structure-only classification — labels and field types, **not** what they typed. |
| **Why it matters** | Submit is a major signal, but **runtime restraint** can still apply. **Demo path progress** may show **Test-drive step** while **Runtime experience depth** in the inspector may still be **Comparing** — that is correct. |

**SAY (short):** “Submit is high intent. Watch the strip advance on the **scripted path** — the inspector is the **runtime judgment**, and it may still withhold a harder escalation.”

**POINT:** Post-submit copy on the page: high intent ≠ automatic extra interruption.

**Do not say:** “Submit earned escalation” or “Now they’re in escalation earned.” Say **Test-drive step** on the strip and read **Runtime experience depth** from the inspector.

---

## 4. Open inspector — 1 minute

**DO:** If not already open: click **Toggle Optiview judgment panel** (SI chip, lower-left) or **Ctrl+Shift+`** (backtick, not **D**). Stay on **Buyer view** unless the room is technical — then offer **Operator view** after.

**SAY:** “This is the judgment a marketer can read without engineering jargon — same session, anonymous.”

Walk top to bottom (do not read every field):

| Section | What to say |
|---------|-------------|
| **Current commercial read** | “Plain-language summary of what this visit looks like commercially right now.” |
| **Runtime experience depth** | “Qualitative depth — **not** the demo strip. This is live runtime judgment.” |
| **Recommended next experience** | “What Optiview would activate next — surface in buyer language.” |
| **Why this recommendation** | “Ties the recommendation to signals you can defend in a meeting.” |
| **Why stronger escalation was withheld** | “The restraint story — what we chose **not** to do and why.” |
| **Commercial intent read** | “Financing uncertainty, comparison, scheduling — milestone-style, not a chat log.” |
| **Timeline** | “Milestones from behavior — not PII, not form values.” |

**CUE (optional):** “Would you trust this read in a weekly ops review?”

**Do not say:** “Escalation earned” unless the inspector’s **Runtime experience depth** actually shows that state. Do not conflate with **Test-drive step** on the demo strip.

---

## 5. Explain restraint — 45 seconds

**SAY:**

> “**High intent does not automatically mean interrupt harder.**  
> If someone is validating financing, building trust, or deciding whether the next step is safe, Optiview can recommend **reassurance** — payment guidance, comparison support, inventory clarity — instead of pressure.  
> We would rather miss a popup than train visitors to ignore noise. **Escalation must be earned** by the full commercial read, not by one hot click.”

**POINT:** **Why stronger escalation was withheld** in the inspector.

**Do not say:** “We always show a popup on convert,” “intent score = show modal,” “the algorithm decided they’re ready to buy.”

---

## 6. Integration close — 45 seconds

**SAY:**

> “You keep your stack. Optiview feeds **decisions** — stable surface IDs and confidence — into **GTM, Adobe Target, Optimizely, AEM, Shopify, or React** headless components.  
> If you have **no** experimentation stack yet, you still get **recommended experiences** and a **surface preview** in the inspector; you choose what renders.  
> **By default, Optiview does not rewrite your CMS** — your team controls rendering, timing, and suppression.”

**DO (optional, technical room):** Switch inspector to **Operator view** → show envelope / surface preview briefly.

**Links to hand off:** [integrations/README.md](./integrations/README.md) · [CMS_ACTIVATION_EXAMPLES.md](./CMS_ACTIVATION_EXAMPLES.md) · [WEBMASTER_INSTALL_ONE_PAGER.md](./WEBMASTER_INSTALL_ONE_PAGER.md)

---

## 7. Close — 15 seconds

**SAY:**

> “The product is **not** visitor identification. It is **commercial judgment for anonymous traffic** — what to show next, when to wait, and when to hold back.  
> Week one of a pilot is about **believable decisions** and a clear surface map, not guaranteed lift. We’ll use a scorecard and your real CTAs.”

**DO:** Mention [PILOT_ONBOARDING.md](./PILOT_ONBOARDING.md) and [PILOT_SUCCESS_SCORECARD.md](./PILOT_SUCCESS_SCORECARD.md) if they are moving to pilot.

---

## Exact phrases (keep these crisp)

**Use often**

- “Anonymous experience decision runtime”
- “Commercial judgment, not identity”
- “Structure only — not what they typed”
- “Your stack decides what renders”
- “Escalation must be earned”
- “Runtime experience depth” (inspector) vs “Demo path progress” (strip)
- “Restraint is a feature, not a bug”

**Avoid**

- “We identify visitors” / “cross-site tracking” / “cookie-based profiles”
- “Auto-personalize every page”
- “AI reads your DOM and rewrites copy”
- “Guaranteed lift in week one”
- “Escalation earned” right after test-drive submit (unless inspector shows it)
- Internal jargon: `surface_id`, `primary_decision`, `envelope`, `collect payload` (buyer room)

## Common objections and answers

| Objection | Answer |
|-----------|--------|
| “Is this another popup tool?” | No — decisions can be inline, deferred, or **null**. Suppression and withholding are first-class. |
| “Do you read form data?” | **No values.** Intent from structure only: field names, types, labels, submit button text. |
| “Do you need cookies?” | **No tracking cookies.** Session-scoped storage on **your** origin. |
| “Will you change our site without us?” | **Not by default.** Recommendations + events; you wire surfaces in GTM, Adobe, CMS, or React. |
| “How is this different from analytics?” | Analytics records **what happened**. Optiview recommends **what should happen next** and explains **why not** something stronger. |
| “We already have Optimizely / Adobe.” | Optiview is the **judgment layer** for anonymous commercial behavior; your tools still **deliver** the experience. |
| “What about GDPR / CCPA?” | Anonymous-by-design posture; no cross-site graph. Point security to [optiview.ai/privacy](https://optiview.ai/privacy) and [SECURITY_PRIVACY_FAQ.md](./SECURITY_PRIVACY_FAQ.md). |
| “How do we measure success?” | Pilot week 1: believable reads, three surface opportunities, install verified — [PILOT_SUCCESS_SCORECARD.md](./PILOT_SUCCESS_SCORECARD.md). Lift follows activation discipline. |
| “The strip says Test-drive step but inspector says Comparing.” | **Correct.** Scripted path progress ≠ runtime permission to escalate. That tension is the demo. |

## Fallback — inspector does not open

| Try | Notes |
|-----|--------|
| Click **Toggle Optiview judgment panel** (bottom-left SI chip) | Primary control |
| **Ctrl+Shift+`** (Windows/Linux) or **⌘+Shift+`** (Mac) | **Backtick** key (above Tab), **not** Ctrl+Shift+D (Chrome bookmarks all tabs) |
| Submit test-drive again | Demo calls `openInspectorAfterSubmit()` — should open **Buyer** view |
| Add `?si_debug=1` to the URL | Opens drawer on load; use **Operator** toggle if you need technical fields |
| Hard refresh once | Clears a stuck drawer state |

**SAY if still stuck:** “The live read still ran in the browser — I’ll show the same sections from a fresh session or send a short screen recording. The product judgment is session-local; this is a demo UI quirk, not your production install.”

---

## Fallback — CDN / stale snippet

Symptoms: old inspector labels, missing **Runtime experience depth**, wrong restraint copy.

| Check | Action |
|-------|--------|
| Demo footer build stamp | Compare commit hash to latest deploy |
| [https://cdn.optiview.ai/version.json](https://cdn.optiview.ai/version.json) | `commit` should match latest release |
| Unversioned `si.js` cache | CDN may cache ~5 minutes — use versioned URL `https://cdn.optiview.ai/si.js?v=<commit>` or hard refresh |
| Demo env pin | Production demo build should set `VITE_SI_SNIPPET_VERSION` to the deployed commit |

**SAY:** “We version the snippet — I’ll load the pinned build so you’re seeing the same judgment UI we ship.”
